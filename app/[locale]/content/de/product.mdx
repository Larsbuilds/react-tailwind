## Warum wird Specify4IT benötigt?

### Code ohne Fehler erstellen
Die Erstellung von "von Anfang an richtiger" Software für große moderne Programme ist fast unmöglich, und Benutzer sind ständig frustriert von der Notwendigkeit, Patches und Updates zu installieren. Dies ist kein einfach zu lösendes Problem, da moderne Programme und Programmiersprachen oft zu komplex sind, um von einzelnen Programmierern vollständig verstanden und überprüft zu werden. Oft können Kunden ihre Anforderungen nicht klar, präzise und eindeutig spezifizieren, was es für den Programmierer schwierig macht, fehlerfreien Code zu erstellen.

## Warum sind bestehende Methoden unzureichend?

### Den Code testen
Für Datenbanksysteme verlassen sich bestehende Methoden auf das Testen des fertigen Codes, um sicherzustellen, dass er der Spezifikation entspricht. Dies kann jedoch hunderte von Stunden Programmierung erfordern, bevor ein Fehler gefunden wird. Die Behebung der Fehler erfordert ebenfalls erhebliche Zeit und Kosten. Selbst dann können Kunden ihre Meinung ändern, wenn sie das System in Aktion sehen!

### Testen ist kein Beweis für Korrektheit
Wir sollten Professor Dijkstras Warnung beachten: "Testen kann die Anwesenheit von Fehlern zeigen, aber niemals deren Abwesenheit".
Tatsache ist, dass Programmierung ein komplexer, arbeitsintensiver und fehleranfälliger Prozess ist. Wenn eine Spezifikation nicht als korrekt bewiesen werden kann, kann auch nicht bewiesen werden, dass die resultierende Software der Spezifikation entspricht.

### Formale Methoden
Eine Alternative wäre die Verwendung formaler Methoden wie Z oder B-Sprachen. Diese sind jedoch wirklich nur im Bereich hochqualifizierter Experten mit formaler mathematischer und logischer Ausbildung angesiedelt.

### Ein besserer Weg
Was wäre, wenn es einen Weg gäbe, eine Software-Spezifikation zu erstellen, die nachweislich korrekt ist, vom Kunden validiert wurde und deren Systemverhalten vor der Codegenerierung überprüft werden könnte - und das alles zugänglich für alle Programmierer?

## Was liefert Specify4IT?

### Mathematischer Beweis der Korrektheit
Dies wird erreicht, indem die Spezifikation auf sehr strukturierte Weise ausgedrückt wird; das System wird als Satz von Prozessen definiert, die durch Regeln verbunden und durch Ereignisse ausgelöst werden. Auf diese Weise wird eine Spezifikation erstellt, die mathematisch auf Konsistenz überprüft werden kann, deren Verhalten bewertet und animiert werden kann, um Korrektheit sicherzustellen, und deren Übereinstimmung mit der Spezifikation bewiesen werden kann. All dies, bevor Code generiert wird.

Specify4IT kann dann diese mathematisch definierte Spezifikation, die vom Kunden vereinbart wurde, verwenden und automatisch fehlerfreien Code generieren.

## Wie macht Specify4IT das?

### Hochinnovativ
Das automatisierte Toolset von Specify4IT verwendet eine neue und innovative axiomatische Beweistheorie für beweisbare Software (genannt Predict), um diese Revolution in der Softwareerstellung zu ermöglichen.
Vor jeder Codegenerierung ermöglicht Specify4IT dem Benutzer:

#### Prüfen
Überprüfen, dass alle Aspekte der Spezifikation konsistent sind und keine Konflikte bestehen.

#### Animieren
Das in der Benutzerspezifikation enthaltene Verhalten animieren, sodass ein Benutzer bewerten kann, was in allen Situationen passiert.

#### Ableiten
Die Anwendungsanforderungen ableiten, zum Vergleich mit den ursprünglichen Anweisungen des Kunden.

#### Beweisen
Die Eigenschaften der Anwendung beweisen.
Animation, Ableitung und Beweis der Spezifikation werden automatisch durchgeführt. Der eigentliche Code für die spezifische Anwendung wird automatisch generiert - in wenigen Minuten.
Specify4IT stellt sicher, dass Spezifikationen Vollständig, Konsistent, Korrekt und Verständlich sind, und dies wurde unabhängig mit dem Imperial College London verifiziert.
